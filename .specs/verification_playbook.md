# Verification Playbook: Career Pilot System Validation

This document provides a step-by-step testing and verification framework to ensure the security, reliability, and correctness of the **career-pilot** system.

---

## 1. Local Bootstrapping & Telegram Bot Test

To verify the natural language onboarding state machine without setting up live Telegram webhook endpoints during local development:

### Setup Simulated Onboarding Script
Create a temporary script `backend/scratch/test-onboarding.ts` that mocks message events:

```typescript
import { processTelegramMessage } from '../src/channels/telegram';
import { db } from '../src/database/sqlite';

async function runOnboardingTest() {
  console.log("Initializing database...");
  await db.migrate();
  await db.run('DELETE FROM candidate_profile');
  await db.run("INSERT OR REPLACE INTO system_settings (key, value) VALUES ('is_bootstrapped', 'false')");

  const steps = [
    { text: "Hello", expectedResponse: /What is your full name/i },
    { text: "Alexander LaGonterie", expectedResponse: /What target roles/i },
    { text: "Senior Software Engineer, AI Developer", expectedResponse: /location preferences/i },
    { text: "Remote, or Hybrid in Austin, TX", expectedResponse: /paste your complete master resume/i },
    { text: "My Resume: Senior Engineer at FinTech Corp. Built agent systems.", expectedResponse: /confirm/i },
    { text: "Yes, looks perfect!", expectedResponse: /bootstrapping complete/i }
  ];

  let chatId = 12345;
  for (const step of steps) {
    console.log(`\nUser: "${step.text}"`);
    const reply = await processTelegramMessage(chatId, step.text);
    console.log(`Bot: "${reply}"`);
    if (!step.expectedResponse.test(reply)) {
      throw new Error(`Onboarding assertion failed! Expected matching: ${step.expectedResponse}`);
    }
  }
  
  console.log("\nSuccess: Onboarding state machine successfully updated database!");
}

runOnboardingTest().catch(console.error);
```

### Verification Command
```bash
npx tsx backend/scratch/test-onboarding.ts
```

## 2. Anonymization Engine Verification

To prevent PII leaks or company disclosures on the public Recruiter Portal, we must run automated tests on the sanitization pipeline.

### Sanitizer Audit Test (`backend/scratch/test-sanitizer.ts`)
```typescript
import { sanitizePublicLog } from '../src/gateway/sanitizer';
import { db } from '../src/database/sqlite';

async function runSanitizationTest() {
  // Populate private applications to check company obfuscation
  await db.run("INSERT OR REPLACE INTO private_applications (id, company_name, job_title, status, last_updated) VALUES (?, ?, ?, ?, ?)", [
    "app_test_123",
    "Google",
    "Staff Engineer",
    "INTERVIEWING",
    new Date().toISOString()
  ]);

  const rawLog = "Applied to Google. Reached out to recruiter alex.l@google.com at 555-0199. Resume sent to 1600 Amphitheatre Pkwy.";
  console.log("Raw Log:", rawLog);

  const sanitizedLog = await sanitizePublicLog(rawLog);
  console.log("Sanitized Log:", sanitizedLog);

  // Assertions
  if (sanitizedLog.includes("Google")) {
    throw new Error("Sanitization Failure: Company name was not obfuscated!");
  }
  if (sanitizedLog.includes("alex.l@google.com")) {
    throw new Error("Sanitization Failure: Email address was not redacted!");
  }
  if (sanitizedLog.includes("555-0199")) {
    throw new Error("Sanitization Failure: Phone number was not redacted!");
  }

  console.log("Success: Sanitizer successfully redacted company and PII!");
}

runSanitizationTest().catch(console.error);
```

## 3. Local Ollama Fallback Mechanics

We must ensure that agent containers fail-safe and fall back to GPU-accelerated local Ollama when OpenRouter/Portkey API is unavailable.

### Fallback Verification Protocol
1. **Simulate Outage:** Unset the `PORTKEY_API_KEY` or modify the Portkey URL in the `.env` configuration to point to a non-existent port (e.g. `http://localhost:9999`).
2. **Execute Agent Execution:** Trigger the tailer or hunter agent script.
3. **Verify Host Logs:**
   - Confirm a warning logs: `Primary model execution failed: ... Triggering Ollama fallback...`.
   - Confirm text is successfully generated by `llama3.2` locally.
   - Confirm the task successfully completes and returns output without throwing an unhandled exception.

## 4. Recruiter Playground Sandboxing

We must verify that visits to the recruiter playground do not pollute the persistent sqlite database.

### Audit Plan
1. **Perform Simulation:** Submit a mock company application via the `/api/simulate` endpoint.
2. **Read Database:** Perform a SELECT query on `private_applications` and `public_audit_trail`.
3. **Assert Separation:** Ensure that no database writes occurred for the mock company transaction.

## 5. Cloudflare Tunnel Verification

To ensure that no traffic bypasses the secure tunnel and the VM firewall is correctly locked down:

### Port Scanning Check
1. From an external machine, scan the VM's public IP:
   ```bash
   nmap -p 80,443,4000,22 <GCP_VM_IP>
   ```
2. **Assertion:** Ports `80`, `443`, and `4000` must report `FILTERED` or `CLOSED`. Port `22` (SSH) should be open (or closed if only utilizing Identity-Aware Proxy (IAP)).

### Tunnel Status Check
1. SSH into the GCP VM.
2. View the `cloudflared` container logs:
   ```bash
   docker logs cloudflared
   ```
3. **Assertion:** Logs must show successful connection registration:
   `Connection registered with edge...` and no egress route mapping failures.
